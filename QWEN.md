# Инструкции для Qwen Code

## Pull Request Оформление

При создании Pull Request ВСЕГДА:

### 1. Заголовок PR
- Должен быть кратким и понятным на русском языке
- Формат: `[Тип изменений]: Краткое описание`
- Примеры:
  - `Добавлена: Оптимизация производительности сжатия`
  - `Исправлена: Ошибка утечки памяти в async_io`
  - `Улучшена: Обработка ошибок в compressor_manager`

### 2. Описание PR
Добавляй подробное описание на русском языке со следующей структурой:

```markdown
## 📋 Описание
Подробное описание того, что было изменено и почему.

## 🔧 Тип изменений
- [ ] 🐛 Исправление ошибки (bug fix)
- [ ] ✨ Новая функциональность (feature)
- [ ] ⚡ Оптимизация производительности (performance)
- [ ] 📝 Документация (documentation)
- [ ] 🔧 Рефакторинг (refactoring)
- [ ] 🧪 Добавлены тесты (tests)

## 📁 Изменённые файлы
- `src/file1.cpp` - краткое описание изменений
- `src/file2.h` - краткое описание изменений

## ✅ Тестирование
Описание проведённого тестирования:
- Какие тесты были запущены
- Результаты тестов
- Известные проблемы (если есть)

## 📸 Скриншоты (если применимо)
Скриншоты или примеры вывода до/после изменений.
```

### 3. Команда для создания PR
После коммита автоматически создавай PR с помощью:
```bash
gh pr create \
  --title "Заголовок на русском" \
  --body "Подробное описание на русском языке" \
  --base main
```

## Сборка и тестирование

### Локальная сборка
```bash
./_build.sh
```

### Удалённая сборка по SSH
По умолчанию все команды сборки и тестирования выполнять на удалённой машине через SSH:

```bash
ssh user@remote-host "cd /path/to/project && ./_build.sh && ./bin/compressor-manager"
```

#### Передача файлов на удалённую машину
Перед сборкой синхронизируй изменения:
```bash
rsync -avz --exclude='.git' ./ user@remote-host:/path/to/project/
```

#### Проверка сборки
После деплоя проверяй:
1. Компиляция: `ssh user@remote-host "cd /path/to/project && make"`
2. Запуск: `ssh user@remote-host "/path/to/project/bin/compressor-manager --test"`
3. Логи: `ssh user@remote-host "tail -f /path/to/project/logs/app.log"`
