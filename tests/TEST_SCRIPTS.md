# Реестр тестовых скриптов

Этот файл ведёт учёт всех тестовых скриптов для повторного использования при тестировании mh-compressor-manager.

## Расположение
Все скрипты хранятся в папке `tests/scripts/`.

## Правило использования
При необходимости выполнить скрипт на удалённой системе (`www`):
1. Создать/изменить скрипт **локально** в `tests/scripts/`
2. Скопировать на сервер: `scp tests/scripts/script.sh www:/tmp/script.sh`
3. Выполнить: `ssh www "bash /tmp/script.sh"`
4. Сохранить финальную версию в репозиторий
5. Обновить этот файл

## Скрипты

### 1. gen_loadtest.sh
- **Назначение:** Генерация нагрузочных тестовых данных (1GB+)
- **Тип:** Генерация данных
- **Статус:** ✅ Готов к использованию
- **Описание:** Создаёт 6 типов файлов (html, css, js, json, xml, txt) в `/srv/123/loadtest/` с предварительным сжатием ~40% файлов для проверки механизма stale.
- **Параметры:**
  - HTML: 80 файлов (~9MB)
  - CSS: 50 файлов (~7MB)
  - JS: 50 файлов (~6MB)
  - JSON: 60 файлов (~49MB)
  - XML: 50 файлов (~20MB)
  - TXT: 80 файлов логов (~1.7GB)
  - Итого: ~1.8GB, 370 файлов
- **Последнее использование:** 2026-04-11, нагрузочное тестирование памяти и initial scan
- **Файл:** `tests/scripts/gen_loadtest.sh`

### 2. loadtest_comprehensive.sh
- **Назначение:** Комплексное нагрузочное тестирование (генерация + сжатие + валидация)
- **Тип:** Нагрузочное + интеграционное тестирование
- **Статус:** ✅ Готов к использованию
- **Описание:** 7 этапов: подготовка → генерация 500 файлов (~346MB) → stale detection (40%) → запуск сервиса с мониторингом → проверка корректности gzip/brotli → стресс-тест (100 файлов за раз) → финальные метрики → очистка
- **Этапы:**
  1. Остановка сервиса, очистка директории
  2. Генерация HTML/CSS/JS/JSON/TXT файлов
  3. Предварительное сжатие 40% gzip + установка старых mtime
  4. Запуск сервиса, мониторинг каждые 30 сек (memory, CPU, прогресс)
  5. Валидация: gzip -t, brotli -t, сравнение размеров распаковки
  6. Стресс-тест: 100 новых файлов за 1 секунду
  7. Итоговая таблица + очистка
- **Результаты первого запуска (2026-04-11):**
  - 500 файлов сгенерировано, 346MB
  - 914 сжатых файлов создано
  - gzip/brotli: 20/20 валидных, 0 ошибок
  - Память: peak 346MB → 8.6MB (утечки нет)
  - 0 реальных ошибок
- **Файл:** `tests/scripts/loadtest_comprehensive.sh`
