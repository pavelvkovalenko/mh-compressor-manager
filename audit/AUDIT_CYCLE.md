# Процедура цикла аудита — mh-compressor-manager

## Режимы аудита

### Режим 1: Цикл аудита (стандартный)

```
ЦИКЛ:
  1. Запустить раунд аудита:
     - Проверить ВСЕ файлы .cpp/.h в src/
     - Если ошибок НЕ найдено → ВЫХОД из цикла
  2. Исправить ОДНУ из самых простых найденных проблем
     - Остальные ошибки НЕ трогать (сохранить консистентность)
  3. Запушить исправление
  4. Сбросить всю логику проверки (память о предыдущих проверках)
  5. Вернуться к пункту 1
```

### Режим 2: Цикл аудита с полным тестированием

```
ЦИКЛ:
  1. Запустить раунд аудита:
     - Проверить ВСЕ файлы .cpp/.h в src/ (полная проверка кода)
     - Если ошибок в коде НЕ найдено → перейти к пункту 2 (тесты)
     - ИНАЧЕ (найдены ошибки в коде):
       → Перейти к пункту 3 (исправление)
  2. Запустить тесты по ОДНОМУ:
     - Запустить один тест → проверить результат
     - Если тест выявил ошибку → перейти к пункту 3 (исправление)
     - Если тест пройден → запустить следующий тест
     - Если ВСЕ тестЫ пройдены → ВЫХОД из цикла
  3. Исправить ОДНУ из самых простых найденных проблем
     (ошибку в коде или в тесте — что проще исправить)
  4. Запушить исправление
  5. Сбросить всю логику проверки (память о предыдущих проверках)
  6. Вернуться к пункту 1 (НОВЫЙ РАУНД)
```

> **Важно:** Режим 2 = Режим 1 + серия тестов (не вместо, а в дополнение).
> Тесты запускаются только если код прошёл проверку без ошибок.
> При любой ошибке (в коде или в тесте) — исправляется одна проблема и цикл перезапускается.

## Критерии выхода из цикла

- **Режим 1 (Аудит):** Все файлы src/ проверены, ошибок не найдено
- **Режим 2 (Тестирование):** Все файлы src/ проверены (без ошибок) И все тесты пройдены

## Правила

1. **ОДНО исправление за раунд** — не массовые изменения
2. **Сброс контекста** — каждый раунд начинается с чистого листа
3. **Журналирование** — каждое изменение фиксируется в CHANGELOG.md
4. **Подсчёт раундов** — ведётся статистика количества раундов
