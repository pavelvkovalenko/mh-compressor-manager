# Процедура «Проверь документацию» — mh-compressor-manager

## Назначение

Перед выполнением любого действия (аудит, тестирование, исправление кода, изменение кода, пуш) —
проверить актуальность документации проекта.

## Обоснование

ТЗ, спецификации, требования и настройки проекта могут быть изменены **вне текущего контекста беседы**:

| Источник изменений | Пример |
|-------------------|--------|
| **Вручную пользователем** | Пользователь отредактировал ТЗ, добавил новые требования |
| **Третьим лицом** | Другой разработчик, менеджер, ревьюер внёс изменения |
| **В другой ветке/PR** | Изменения в другой ветке обсуждения или Pull Request |
| **Через файлы .md** | Любое изменение файлов документации вне текущего сеанса |

**Риск:** Выполнение действий на основании устаревших требований приведёт к:
- Созданию кода, не соответствующего актуальным требованиям
- Нарушению логики, определённой в обновлённом ТЗ
- Массовым переделкам и ошибкам

## Процедура

### Шаг 1: Проверка изменений в файлах .md

```bash
cd /path/to/project
git status --short | grep '\.md$'
git diff --name-only HEAD | grep '\.md$'
```

**Файлы для обязательной проверки:**
- `docs/specification/TECHNICAL_SPECIFICATION.md` — ТЗ и требования
- `docs/development/QWEN.md` — правила и контекст AI-ассистента
- `docs/development/RULES.md` — правила написания кода
- `docs/development/DEPLOY.md` — сборка и деплой
- `CONTRIBUTING.md` — Pull Request и коммиты
- `README.md` — обзор проекта
- `audit/CHANGELOG.md` — журнал изменений
- `audit/ROUND_HISTORY.md` — история раундов аудита
- `tests/TEST_SCRIPTS.md` — реестр тестовых скриптов

### Шаг 2: Если изменения обнаружены

1. **Прочитай изменившиеся файлы** полностью
2. **Определи характер изменений:**
   - Новые требования к функциональности
   - Изменения в логике поведения
   - Обновлённые настройки проекта
   - Новые правила написания кода
3. **Загрузи изменения в контекст текущей беседы:**
   - Перечитай содержимое изменённых файлов
   - Запомни новые требования
   - Скорректируй план действий с учётом изменений
4. **Уведоми пользователя:**
   ```
   📄 Обнаружены изменения в документации:
   - Файл: TECHNICAL_SPECIFICATION.md (изменены требования к сжатию)
   - Файл: QWEN.md (добавлено правило проверки документации)
   Учтено в текущей работе.
   ```

### Шаг 3: Если изменений нет

Продолжай выполнение запланированного действия.

## Критерии завершения

- [ ] Все файлы `.md` проверены через `git status` и `git diff`
- [ ] Изменённые файлы прочитаны полностью
- [ ] Новые требования учтены в плане действий
- [ ] Пользователь уведомлён об изменениях (если есть)
