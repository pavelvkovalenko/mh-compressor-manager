1. Обзор

mh-compressor-manager — это фоновая служба (демон) для автоматического сжатия статического контента веб-сервера nginx. Программа отслеживает изменения в файловой системе в реальном времени и создает сжатые копии файлов в форматах Gzip (.gz) и Brotli (.br).

✅ Цель: Снижение сетевого трафика и ускорение загрузки веб-страниц за счет предварительного сжатия статических ресурсов.

2. История разработки

📜 Историческая справка:

Программа mh-compressor-manager версии 1.0.0 была разработана в 2026 году специально для операционной системы Fedora 43.

Предыдущая версия утилиты сжатия была написана в 2014 году и имела следующие ограничения:
  • Поддерживался только один алгоритм сжатия — Gzip (формат .gz)
  • Отсутствовала система журналирования событий (логов)
  • Не было поддержки многопоточности и асинхронной обработки
  • Отсутствовала интеграция с systemd
  • Конфигурация осуществлялась только через аргументы командной строки
Текущая версия полностью переработана с использованием современных стандартов (C++23) и учитывает все требования к надежности, безопасности и производительности.

3. Возможности

  • Мониторинг в реальном времени — использование inotify для отслеживания событий файловой системы
  • Два алгоритма сжатия — Gzip (zlib) и Brotli (libbrotli)
  • Рекурсивное сканирование — поддержка вложенных директорий
  • Многопоточность — пул потоков для параллельного сжатия файлов
  • Debounce-защита — предотвращение множественной обработки при частых изменениях
  • Идемпотентность — пропуск файлов, которые уже сжаты и актуальны
  • Сохранение метаданных — права доступа, владелец, группа, временные метки, SELinux-контекст
  • Интеграция с systemd — уведомления о готовности, корректное завершение работы
  • Гибкая конфигурация — INI-файл + аргументы командной строки
  • Подробное логирование — интеграция с syslog/journald с уровнями DEBUG/INFO/WARNING/ERROR

5. Установка

# 1. Сборка RPM-пакета (из исходников)
cd /path/to/mh-compressor-manager
./build-rpm.sh 1.0.0

# 2. Установка собранного пакета
sudo dnf install ~/rpmbuild/RPMS/x86_64/mh-compressor-manager-1.0.0-1*.rpm

# 3. Обновление базы systemd
sudo systemctl daemon-reload

5.2. Сборка из исходных кодов

# 1. Установка зависимостей (см. раздел 4.2)

# 2. Клонирование репозитория (или распаковка архива)
cd /path/to/source

# 3. Создание директории сборки
mkdir build && cd build

# 4. Конфигурация проекта
cmake .. -DCMAKE_BUILD_TYPE=Release

6. Конфигурация

6.1. Файл конфигурации

Путь по умолчанию: /etc/mediahive/compressor-manager.conf

[general]
# Каталоги для мониторинга (разделитель ; или пробел)
target_path=/var/www/html;/var/www/site1

# Логирование отладки (true/false)
debug=false

# Количество потоков (0 = автоопределение по CPU)
threads=0

# Список расширений (без точек, через пробел)
list=txt js css svg json html htm map

# Алгоритмы сжатия: gzip, brotli, all
algorithms=all

# Уровень сжатия Gzip (1-9)
gzip_level=6

# Уровень сжатия Brotli (1-11)
brotli_level=4

# Задержка перед сжатием после изменения файла (сек)
debounce_delay=2

# Опционально: переопределение настроек для конкретной папки
[/var/www/site2]
list=html htm
algorithms=all
gzip_level=6
brotli_level=4

6.2. Описание параметров

Параметр Тип По умолчанию Описание
target_path string /var/www/html Каталоги для мониторинга
debug boolean false Режим отладочного логирования
threads integer 0 (авто) Количество потоков пула
list string txt js css... Список расширений файлов
algorithms string all Алгоритмы: gzip, brotli, all
gzip_level integer 6 Уровень сжатия Gzip (1-9)
brotli_level integer 4 Уровень сжатия Brotli (1-11)
debounce_delay integer 2 Задержка перед сжатием (сек)
⚠️ Важно: Аргументы командной строки имеют приоритет над настройками в файле конфигурации.

7. Командная строка

Аргумент Описание Пример
--config <path> Путь к файлу конфигурации --config /etc/my.conf
--dir <path> Целевая директория (переопределяет config) --dir /var/www/html
--ext <list> Список расширений (переопределяет config) --ext "js css html"
--gzip-level <N> Уровень сжатия Gzip (1-9) --gzip-level 9
--brotli-level <N> Уровень сжатия Brotli (1-11) --brotli-level 11
--dry-run Режим проверки (без реального сжатия) --dry-run
--help, -h Вывод справки --help
--version, -v Вывод версии программы --version

Примеры использования CLI

# Запуск с настройками по умолчанию
sudo mh-compressor-manager

# Запуск в режиме отладки с конкретной директорией
sudo mh-compressor-manager --debug --dir /var/www/mysite

# Тестовый запуск без реального сжатия
sudo mh-compressor-manager --dry-run --debug

# Запуск с альтернативным конфигом
sudo mh-compressor-manager --config /opt/custom.conf

# Запуск с переопределением уровней сжатия
sudo mh-compressor-manager --gzip-level 9 --brotli-level 11

8. Интеграция с systemd

8.1. Файл юнита

[Unit]
Description=MediaHive Compressor Manager
After=network.target

[Service]
Type=notify
ExecStart=/usr/bin/mh-compressor-manager --config /etc/mediahive/compressor-manager.conf
Restart=on-failure
User=root
Group=root

[Install]
WantedBy=multi-user.target

8.2. Управление службой

# Запуск службы
sudo systemctl start mh-compressor-manager

# Остановка службы
sudo systemctl stop mh-compressor-manager

# Перезапуск службы
sudo systemctl restart mh-compressor-manager

# Проверка статуса
sudo systemctl status mh-compressor-manager

# Включение автозапуска при загрузке
sudo systemctl enable mh-compressor-manager

# Отключение автозапуска
sudo systemctl disable mh-compressor-manager
📌 Тип службы: Type=notify — программа отправляет уведомление о готовности через sd_notify().

11. Решение проблем

11.1. Служба не запускается

# Проверка статуса службы
sudo systemctl status mh-compressor-manager

# Просмотр логов на ошибки
sudo journalctl -u mh-compressor-manager -p err

# Проверка конфигурации
sudo mh-compressor-manager --config /etc/mediahive/compressor-manager.conf --dry-run

11.2. Файлы не сжимаются

  • Проверьте права доступа к целевой директории
  • Убедитесь, что расширение файла есть в списке list
  • Проверьте логи на наличие ошибок
  • Запустите в режиме --debug для подробной диагностики

11.3. Превышение лимита inotify

# Проверка текущего лимита
cat /proc/sys/fs/inotify/max_user_watches

# Увеличение лимита (временно)
sudo sysctl fs.inotify.max_user_watches=524288

# Увеличение лимита (постоянно)
echo "fs.inotify.max_user_watches=524288" | sudo tee -a /etc/sysctl.conf
sudo sysctl -p

11.4. Ошибки прав доступа

  • Запускайте службу от root для доступа к системным директориям
  • Проверьте права на запись в целевую директорию
  • Убедитесь, что SELinux не блокирует доступ (проверьте setenforce 0 для теста)

11.5. Высокое потребление памяти

  • Уменьшите количество потоков в конфигурации (threads=2)
  • Проверьте размер обрабатываемых файлов
  • Увеличьте debounce_delay для снижения нагрузки

11.6. Ошибки сборки на Fedora 43

⚠️ Fedora 43: zlib-ng-compat-devel

Если при сборке возникает ошибка «zlib-devel not found», установите актуальный пакет: 
sudo dnf install zlib-ng-compat-devel
Также обновите CMakeLists.txt, заменив pkg_check_modules(ZLIB REQUIRED zlib) на pkg_check_modules(ZLIB REQUIRED zlib-ng-compat) при необходимости.

12. Безопасность

12.1. Права доступа

  • Конфигурационный файл: 644 (владелец root)
  • Бинарный файл: 755 (владелец root)
  • Рекомендуется запуск от специализированного пользователя mh-compressor

12.2. Защита от уязвимостей

  • ❌ Запрещено выполнение внешних команд (system(), popen())
  • ❌ Игнорирование символических ссылок (защита от symlink-атак)
  • ✅ Использование lchown() вместо chown() для безопасности
  • ✅ Проверка прав доступа перед операциями записи

12.3. SELinux

При включенном SELinux сжатые копии наследуют контекст безопасности оригинала. Для диагностики проблем с SELinux:

# Проверка контекста файла
ls -Z /var/www/html/file.js

# Просмотр логов SELinux
sudo ausearch -m avc -ts recent

# Временное отключение (для тестирования)
sudo setenforce 0
